## Versioning

This project uses major.minor.revision version numbers.

- Major: breaking or foundational application changes.
- Minor: meaningful feature additions or workflow expansions.
- Revision: routine changes, fixes, styling updates, and small improvements.

Each committed project change should increment the revision number unless the change intentionally bumps the major or minor version.

Every versioned change should also be tagged in Git with the matching version number, such as `v1.1.3`.

Revision details are tracked in `revision.md` with newest versions first.
